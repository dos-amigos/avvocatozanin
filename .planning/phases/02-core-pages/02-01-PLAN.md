CRITICAL implementation notes:
- Every animation block must null-check its target element(s) before animating: `var cards = gsap.utils.toArray('.value-card'); if (!cards.length) return;` (use early return per block, not for the whole IIFE)
- Use `once: true` on all non-parallax ScrollTriggers (parallax uses `scrub: true` which is inherently not "once")
- The splitWords function is available via `window.splitWords` (exported from animations.js which loads before home.js)
- For parallax elements (.about-split__img, .cta-section__bg, .stats-section__bg), set `overflow: hidden` on their parent container via CSS if not already set (check: .about-split__media has no overflow hidden -- the img scaling will bleed. Add `will-change: transform` to parallax targets for GPU compositing)
- Do NOT use gsap.from on elements that are in the viewport on page load without checking -- value-card, about-split etc. are below the fold so gsap.from is fine. Hero elements are above the fold but use a timeline with delay.
