**1. Enable page cache in config.php (PERF-03):**

Update the `'cache'` key in `site/config/config.php` from:
```php
'cache' => [
    'pages' => [
        'active' => false,
    ]
],
```
To:
```php
'cache' => [
    'pages' => [
        'active' => true,
        'ignore' => function ($page) {
            return $page->id() === 'contatti';
        },
    ],
],
```

This enables Kirby's static HTML page cache for all pages EXCEPT /contatti (which has a form and must render dynamically to show validation errors/success state).

**2. Audit all templates for lazy loading (PERF-01):**

For each template, check every `<img>` and `<iframe>` tag:

**Rules:**
- Hero/LCP images (first viewport): `loading="eager"` or NO loading attribute (browser default is eager)
- All other images below the fold: `loading="lazy"`
- All `<iframe>` elements (Maps): `loading="lazy"`
- Video `<poster>` attributes do NOT get loading — that's an img-only attribute

**Expected findings per template:**

- `home.php`: Hero video/image should be `loading="eager"`. CTA background image should be `loading="lazy"`. Stats section images (if any) should be `loading="lazy"`.
- `servizi.php`: Hero image `loading="eager"`. Service card images (if any) `loading="lazy"`.
- `servizio.php`: Hero video poster is eager (correct — already has `loading="eager"` on the fallback img). CTA bg image already has `loading="lazy"` (correct). No changes expected.
- `chi-sono.php` (from Plan 01): Hero bg image `loading="eager"`. Portrait in bio section `loading="lazy"`. Values section icons are Lucide (not img).
- `come-lavoro.php` (from Plan 01): Hero bg image `loading="eager"`. No other images expected.
- `contatti.php` (from Plan 02): Hero bg image `loading="eager"`. Maps iframe `loading="lazy"`. CTA bg image `loading="lazy"`.

For each template, add or fix `loading` attributes as needed. Do NOT modify files that are already correct.

**3. Thumb presets reference check (PERF-02):**

The existing templates use Unsplash URLs (external images), not Kirby-managed files. Thumb presets (`->thumb('card')`, `->thumb('hero')`) only apply to Kirby file objects (`$page->image()`, `$file->thumb()`).

For Phase 3 pages:
- Chi Sono portrait: if the blueprint has a file upload field AND an image is uploaded, the template should use `->thumb('card')`. If using Unsplash URL (the current approach), no thumb preset applies — just render the URL directly.
- The thumb presets are already correctly configured in config.php. No changes needed unless a template uses Kirby file objects without applying a preset.

Document in the SUMMARY that thumb presets are configured and will be applied when real uploaded images replace Unsplash placeholders.
